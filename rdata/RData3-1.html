3.1 ინსტალაცია. პაკეტები

      დროა გავეცნოთ R-ის საწყისებს.  R - დაპროგრამების ენაა, რომელიც გარკვეული სპეციფიკურობით ხასიათდება - ის იმავდროულად სტატისტიკური გაანგარიშებებისა და გრაფიკული ანალიზის  გარემოცაა.  როგორც უკვე ავღნიშნეთ R  უსასყიდლოთ ვრცელდება.  მისი გადმოწერა და დაინსტალირება შესაძლებელია R-ის სრული ქსელური არქივიდან (Comprehensive R Archive Network, CRAN), რომლის მისამართიცაა:

http://cran.r-project.org

ოპერაციული სისტემის მიხედვით გადავიდეთ საინსატალაციო პაკეტების გვერდზე და ავირჩიოთ: install R for the first time.  იქვეა მოცემული ინსტრუქცია თუ როგორ უნდა დავაინსტალიროთ ის, სხვადასხვა ოპერაციული სისტემებისათვის. R-ის ფუნქციონალი არ განსხვევდება ოპერაციული სისტემების მიხედვით. ოპერაციული სისტემისათვის დამახასიათებელი თავისებურებები, მცირედი სხვაობებით, ვლინდება ინტერფეისთან და ოპერაციულ სისტემასთან მუშაობასას დაკავშირებულ საკითხებში. შემდგომში ჩვენ საუბარი გვექნება Windows ოპერაციული სისტემისათვის მოყვანილ  მაგალთებზე, თუმცა სხვა ოპერაციული სისტემებისათვის სპეციფიკურ საკითხებზე ინფორმაცია მოიძიება ამ პროგრამების საიტებზე. ინსტალაციის შემდგომ შეგვიძლია გავუშვათ R-ის გრაფიკული ინტერფეისი და განვაგრძოთ მუშაობა მასში.  თუმცა უმჯობესია შემდგომი მუშაობის გამარტივებისათვის გადმოვწეროთ და დავაინსტალიროთ  R-ის ე. წ. გარსული პროგრამა - დამუშავების ინტეგრირებული გარემო (Integrated Development Environment -IDE) Rstudio, რომელიც ასევე უფასო ლიცენზიით ვრცელდება და ხელმისაწვდომია შემდეგ მისამართზე:

http://www.rstudio.com/ide/

ავირჩიოთ ღილაკი Download Rstudio Desktop. როდესაც ორივე პროგრამა დაინსტალირებულია დავიწყოთ პირველი გაცნობა R-თან. გავუშვათ Rstudio - მომავალში R-თან მუშაობის ყოველ სესიას ამ პროგრამის მეშვეობით ვაწარმოებთ. განვიხილოთ პროგრამის მიერ  გამოტანილი ეკრანი.


             პროგრამას   ინტუიციური ინტერფეისი აქვს. ის ოთხ არედ არის დაყოფილი, თავზე პროგრამას აქვს მენიუს ზოლი, რომლის ზოგიერთი ხშირად მოხმარებადი პუნქტი დუბლირებულია პიქტოგრამის სახით მის ქვემოთ. პირველი არე მოიცავს პროგრამის (სკრიპტის) ტექსტს, აქ ვახდენთ მის აკრებას, რედაქტირებას, შენახვას და შესრულებისათვის ბრძანების მიცემას. ახალი სკრიპტის დასაწყებად ვხსნით ახალ გვერდს მენიუს მეშვეობით: File - New File - R Script.  პროგრამის თითოეული სტრიქონი ინომრება ავტომატურად, რაც მის გამართვას, შეცდომების ძიებას გვიადვილებს.   მეორე არე - კონსოლის არეა. R-ის კონსოლი (Console) ბრძანებების დაუყონებლივ შესასრულებლად გამოიყენება. ბრძანება აიკრიბება კონსოლის მოწვევის კურსორთან:  > | და სრულდება Enter კლავიშით. ეს აქტუალურია მისი კალკულატორის რეჟიმში გამოყენების, დახმარების მოძიების, რაიმე სასწრაფო ბრძანების პროგრამის ტექსტის მიღმა შესასრულებლად და სხვა შემთხვევებში. პროგრამის ველისაგან განსხვავებით აქ ბრძანებები შეიყვანება და სრულდება ცალ-ცალკე. ამავე კონსოლში აისახება პროგრამის შესრულების, გაანგარიშებების  შედეგები (გარდა გრაფიკული გამოსახულებებისა). კონსოლის არე, მის თავზე მოთავსებულ ჩანართზე დაწკაპუნებით, შეგვიძლია გადავრთოთ ტერმინალის რეჟიმში (Terminal). ტერმინალი ხსნის ოპერაციულ სისტემასთან სამუშაო სესიას. ამდენად, პროგრამიდან გამოუსვლელად შეგვიძლია განვახორციელოთ ის ოპერაციები, რომლებიც ოპერაციულ სისტემასთან სამუშაოდ დაგვჭირდება. ოპერაციულ სისტემასთან მუშაობის გარკვეულ ფუნქციონალს (ფაილებთან, ფოლდერებთან მუშაობა და სხვა) R-ის შიდა  ფუნქციებიც გვაძლევს. მესამე არე რამოდენიმე ჩანართს შეიცავს. პრაქტიკულად ხშირად გამოყენებადია ჩანართი Environment, სადაც წარმოდგენილია პროგრამის შესრულების შედეგად შექმნილი ცვლადები (მათ შორის მონაცემთა ცხრილებიც), მათთან დაკავშირებული, გაანგარიშებული მნიშვნელობები, პროგრამაში განსაზღვრული და შესრულებული ფუნქციები. ამ ჩანართის დახმარებით ძალზე მოსახერხებელია პროგრამის შესრულების ან გამართვის(შეცდომების ძიების)  მომენტში ცვლადების შუალედური კონტროლი, ასევე შეიძლება ზოგ ცვლადზე დაწკაპუნებით მისი სტრუქტურის აქვე დათვალიერება ან მათი აქტუალური, მიმდინარე შიგათავსის, შემცველობის (მაგალითად ცხრილებისათვის) პირველ არეში ახალ ცალკე ჩანართზე დათვალიერება. ცვლადთა გარემოს არის გასასუფთავებლად (მაგალითად პროგრამის ხელახლა შესრულების დროს) საკმარისია დავაწკაპუნოთ არეს თავზე მოთავსებულ ცოცხის პიქტოგრამზე. ამგვარი მეთოდი შესაბამისი არის გასასუფთავებლად  სხვა ჩანართებშიც გამოიყენება.  მოკლედ ავღწერთ დანარჩენ ორ ჩანართსაც: History ჩანართი შეიცავს ადრე მიცემული ბრძანებების ნუსხას, ხოლო   Connections ჩანართი საშუალებას გვაძლევს იოლად მივუერთდეთ სხვადსხვა მონაცემთა წყაროებს და გამოვიკვლიოთ იქ არსებული მონაცემები. ყველაზე მეტ ჩანართს მეოთხე არე შეიცავს. პირველი ჩანართი -  Files ასახავას კომპიუტერში არსებული ფოლდერების სტრუქტურას თავისი ფაილებით. აქვე შეგვიძლია ვაწარმოოთ ოპერაციები ფოლდერებსა და ფაილებზე (წაშლა, კოპირება,  გადატანა და სხვა). Plots ჩანართი გვაჩვენებს პროგრამის შესრულების შედეგად (თუ ის ასეთ ბრძანებებს შეიცავდა) დამუშავებულ და მომზადებულ გრაფიკულ ინფორმაციას (გრაფიკები, დიაგრამები და სხვა). აქვეა შესაძლებელი ამ მასალის ექსპორტი (შენახვა) სხვადსახვა ფორმატებში. ჩანართი Packages გვაჩვენებს სისტემაში დაინსტალირებული პაკეტების სიას, შესაძლებელია ახალი პაკეტის დაინსტალირება, არსებული პაკეტების მიერთება ან განახლება, მათზე ინფორმაციის მოპოვება. R-ის ძირითადი პროგრამა მოიცავს უამრავ ფუნქციას, თუმცა ყველა არსებული ფუნქციის წინასწარ განსაზღვრა და პროგრამაში გათვალისწინება შეუძლებელია. დრო და დრო  მუშავდება ახალი სტატისტიკური, მონაცემთა დამუშავების მეთოდები, იქმნება ახალი ფუნქციები, რომლებიც ამარტივებს ამოცანის გადაწყვეტას ან სპეციალურადაა შექმნილი რაიმე ვიწრო დარგის ამოცანათა შესასრულებლად. ამ პროცესის გასაადვილებლად არსებობს R-ში პაკეტების დაინსტალირებისა და მიერთების მექანიზმი. შექმნილი ათასობით პაკეტიდან ჩვენ შეგვიძლია საჭირო პაკეტი თავისი ფუნქციებით დავაინსტალიროთ მარტივად და პროგრამაში მისი მიერთების შემდეგ გამოვიყენოთ მთელი მისი შესაძლებლობები, ფუნქციები ჩვენი პროგრამის რეალიზაციის პროცესში. ახალი პაკეტების დამმუშავებელი პროგრამისტები ათავსებენ მათ R-ის ყოვლისმომცველ ქსელურ არქივში (CRAN), საიდანაც მხოლოდ პაკეტის სათაურის მითითებით შეგვიძლია გამოვიყენოთ მისი შესაძლებლობები. R-ის ძალა მის მოქნილობაში,  იოლად გაფართოებაშია. გარდა ამ ჩანართიდან, ახალი პაკეტის დაინსტალირება კონსოლის არეში ბრძანების აკრებითაცაა შესაძლებელი.  ამისათვის კონსოლში მოწვევის კურსორთან უნდა ავკრიბოთ ბრძანება install.packages()  და სასურველი პაკეტის სახელი და დავაწვეთ Enter კლავიშს. პაკეტი ავტომატურად დაინსტალირდება. მაგალითად:

install.packages(”პაკეტის სახელი”)

install.packages(“data.table”)     

install.packages(“ggplot2”)

პაკეტის დაინსტალირება სისტემაში ხდება ერთჯერ. მისი გამოყენების წინ აუცილებელია  ჩვენი  პროგრამის ტექსტში მოვახდინოთ პაკეტის მიერთება library() ბრძანების  მეშვეობით. მაგალითად:

library(პაკეტის სახელი)

library(ggplot2)

library(dplyr)

ეს ბრძანება აიკრიბება პროგრამის ტექსტის არეში და შესრულდება ჩვენს პროგრამასთან ერთობლივად. პაკეტის მიერთება პროგრამასთან საჭიროა R-თან მუშაობის ყოველი ახალი სესიის დაწყების დროს, მაშინ როცა ერთჯერ დაინსტალირებული პაკეტი უკვე სისტემაში იქნება. მიერთების შემდეგ პროგრამის ტექსტში შეგვიძლია თავისუფლად გამოვიყენოთ პაკეტის ყველა ფუნქცია. უცნობი პაკეტის დანიშნულებისა და მასში შემავალი ფუნქციების შესახებ ინფორმაცია შეგვიძლია მოვიპოვოთ შემდეგი ბრძანებით:
