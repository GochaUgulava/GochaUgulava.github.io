3.2 პრაქტიკული ნაბიჯები
ჩაშენებული დატასეტები

შევისწავლოთ რამოდენიმე მარტივი ბრძანება, რომლებიც მონაცემთა დათვალიერების, მათ შესახებ ძირითადი ინფორმაციის მიღების საშუალებას მოგვცემს.

ჩვენ უკვე განვიხილეთ კონსოლში ბრძანებების შეყვანა და შესრულება. როგორც ავღნიშნეთ კონსოლი მოსახერხებელია ერთჯერადად ბრძანების შესასრულებლად. თუ საჭიროა შევქმნად მრავალჯერადი გამოყენების სკრიპტი (პროგრამა), ან ბრძანებები და ფუნქციები კვლავ დაგვჭირდება პროგრამის გამართვის, მონაცემთა დამუშავება-შერჩევის, რაიმე მეთოდის სასინჯად ან მომავლისთვის შესანახად უმჯობესია ტექსტი პროგრამს სახით ჩამოვაყალიბოდ. ის შეგვიძლია შესასრულებლად გადავცეთ როგორც ერთიანი მოცულობით, ასევე შევასრულოთ სტრიქონ-სტრიქონ, ან სულაც შერჩეული სტრიქონების მიხედვით.  გავხსნათ ახალი სკრიპტი მენიუს მეშვეობით: File - New File - R Script. შევინახოთ ის ახალი სახელით, მენიუში: File – Save. შემდგომ, ცვლილებების გაკეთების კვალდაკვალ, არ დაგვავიწყდეს მისი შენახვა მენიუს ან სკრიპტის არის თავში მოცემული პიქტოგრამის მეშვეობით. პროგრამის ყოველი ბრძანება ახალი სტრიქონიდან იკრიბება, რომელზე გადასვლაც Enter კლავიშით ხდება. თუ რომელიმე ცალკე სტრიქონის შესასრულებლად გაშვება გვინდა, ვათავსებთ კურსორს ამ სტრიქონზე და ვაწვებით ერთდროულად Ctrl + Enter კლავიშებს. ამავე მიზნისათვის შეგვიძლია მოვნიშნოთ მაუსით პროგრამის ტექსტში შესასრულებელი სტრიქონები ( ან ერთი სტრიქონი) და სკრიპტის არეს თავში განთავსებულ ისრიან პიქტოგრამას Run დავაწკაპუნოთ. პროგრაის გაშვების ფართო შესაძლებლობებია ასევე მოცემული მენიუში: Code – Run Selected Line(s)  ან  Re-Run Previous, Run Region. შემდგომი პრაქტიკული მაგალითები ავკრიბოთ სკრიპტის არეში და სტრიქონ-სტრიქონ შევასრულოთ Ctrl + Enter გამოყენებით, თუ პირდაპირ სხვა რამ არ იქნა მითითებული.

როგორც ავღნიშნეთ, მონაცემთა დამუშავებისათვის ჯერ საჭიროა მათი R-ის გარემოში მოქცევა, მონაცემთა წაკითხვა, შეყვანა. ამის შესახებ მომავალში ვისაუბრებთ, ამჯერად განვიხილოთ პროგრამაში უკვე ჩაშენებული მონაცემები. მათ დასათვალიერებლად ავკრიბოთ და შევასრულოთ შემდეგი ბრძანება:
data()
შედეგად სკრიპტის არეში გახსნილ ახალ ჩანართში (R data sets) გამოტანილი იქნება R-ის ჩაშენებული მონაცემთა ნაკრებების (დატასეტების) ჩამონათვალი თავიანთი მოკლე აღწერით.



უფრო მეტი ინფორმაცია შერჩეული დატასეტის შესახებ შეგვიძლია მოვიპოვოთ დახმარების მოძიების ფუნქციის საშუალებით. მაგალითად:

help(iris)

help(mtcars)

იგივე შედეგს მივიღებთ თუ კონსოლში ავკრიბავთ პაკეტის დასახელებას  კითხვის ნიშნით წინ. ეს მეთოდი ასევე ნებისმიერი ფუნქციის შესახებ კონსოლიდან დახმარების მოპოვებისათვის გამოიყენება:

? ფუნქცია(ან ობიექტი)

?iris

?mtcars

?head()

?summary()

როგორც ვხედავთ ფუნქციებს აუცილებლად მრგვალი ფრჩხილები მოსდევს, ხშირად მათში ფუნქციის არგუმენტებია განთავსებული. ამა თუ იმ ფუნქციის დახმარების არეში შეგვიძლია ვიხილოთ თუ რისთვის გამოიყენება ეს ფუნქცია, რა არგუმენტები ესაჭიროება, რომელია მათგან აუცილებელი, გავეცნოთ ფუნქციის გამოყენების მაგალითებს.

   ვიდრე დავათვალიერებდეთ ან დავამუშავებდეთ მონაცემებს საჭიროა მათი ჩატვირთვა. ქვემოთ მოყვანილი კოდი ჩატვირთავს მონაცემთა ნაკრებს mtcars, რომელიც 32 მოდელის ავტომობილის სხვადასხვა მახასიათებლებს შეიცავს. შემდეგ გამოვიტანთ ამ დატასეტის პირველ ექვს  სტრიქონს:

#working with mtcars dataset

data(mtcars)

head(mtcars)

თითოეულ სტრიქონში მოცემულია ავტომობილის მოდელები, ხოლო სვეტებში მათი სხვადასხვა მახასიათებლები.

    კოდის სტრიქონი ან სტრიქონის ნაწილი, რომელიც # სიმბოლოთი იწყება არ შესრულდება, ის საინფორმაციო ხასიათისაა, შეიცავს კომენტარს (მოკლე აღწერა, ინფორმაცია ცვლადების, ფუნქციების შესახებ ან სხვა)  და საჭიროა იმისათვის ვინც ეცნობა კოდის ტექსტს. როგორც შევხედეთ ფუნქციას head() გამოაქვს დატასეტის პირველი ექვსი სტრიქონი, რაც მოსახერხებელია დიდი მონაცემთა ნაკრებების თავსართის სანახავად. დატასეტის სტრუქტურის - ანუ თუ რა მონაცემებს, ცვლადებს შეიცავს დატასეტი, რა ტიპისაა თითოეული - გამოვიყენოთ შემდეგი კოდი:

str(mtcars)

აქედან ჩანს, რომ დატასეტი დატა ფრეიმია რომელიც 32 ობიექტს (ანუ ავტომობილის მოდელს - სტრიქონს) და 11 ცვლადს (ანუ მახასიათებელს - სვეტს)  შეიცავს. (იმის გასაგებად, თუ რომელი სვეტი რას აღნიშნავს, შეგვიძლია გამოვიძახოთ დახმარება აღნიშნული დატასეტისათვის.) შემდეგ მოცემულია თითოეული ცვლადის დასახელება, მასში არსებული მონაცემების ტიპი და პირველი რამოდენიმე მონაცემი. იგივე შეიძლება გავაკეთოთ მესამე არეში Environment ჩანართის მეშვეობით. მონაცემთა ჩამოტვირთვისთანავე ამ ჩანართში გაჩნდა ჩამოტვირთული დატასეტის დასახელების სტრიქონიც,  დასახელების წინ ლურჯ სამკუთხედზე დაწკაპუნებით იქვე ჩამოიშლება დატასეტის სტრუქტურა, ხოლო დასახელებაზე დაწკაპუნებით პირველ არეში ახალ ჩანართში აისახება მთლიანი მონაცემთა ცხრილი.

        არსებობს ცალკე ფუნქციები მონაცემთა ნაკრებში სტრიქონებისა და სვეტების რაოდენობის გასაგებად.  სკრიპტში ავკრიბოთ და შევასრულოთ შემდეგი კოდი:

nrow(mtcars)    # number of rows in dataset

ncol(mtcars)    # number of columns in dataset

      და ბოლოს დავათვალიეროთ ჩვენი დატასეტის მონაცემთა კრებსითი   მაჩვენებლები (მინიმალური, მაქსიმალური, საშუალო და სხვა მაჩვენებლები) თითოეული სვეტის (ცვლადის)  მიხედვით:

summary(mtcars)

სამუშაო სესიის დასრულების წინ შევინახოთ ჩვენი სკრიპტი, რომლშიც მოიყარა თავი ყველა იმ ბრძანებამ რაც შევასრულეთ:

 ამრიგად,  გადავდგით პირველი პრაქტიკული ნაბიჯები მონაცემთა შესწავლა-დამუშავების გზაზე. გავიგეთ თუ როგორ ხდება დახმარების გამოძახება სხვადასხვა დატასეტების და ფუნქციების აღწერილობის გასაცნობათ (რამოდენიმე გზით), როგორ აისახება კომენტარები პროგრამაში, რა არის ჩაშენებული დატასეტები და როგორ ჩამოვტვირთოთ ისინი. ასევე გავეცანით რამოდენიმე ფუნქციას, ვნახეთ მათი შესრულების შედეგები.

   ჩვევების გასამყარებლად იგივე პროცედურები განახორციელეთ და გაეცანით შედეგებს სხვა დატასეტებისათვისაც (მათი სიის სანახავად - data() ფუნქცია დაგვეხმარება),  დავიწყოთ  iris დატასეტით.

   და ბოლოს - ბონუსი! ჩვენ ჯერ არ გვინახია R-ის მძლავრი გრაფიკული შესაძლებლობები. მათი მცირე ნაწილის სადემონსტრაციოდ კონსოლში აკრიბეთ demo(graphics)  და Enter კლავიშზე დაწკაპუნებით თანდათანობით დაათვალიერეთ მეოთხე არეში Plot ჩანართში გამოტანილი გრაფიკული ნიმუშები (არ იჩქაროთ, ზოგჯერ ნახაზის ჩატვირთვას გარკვეული დრო ჭირდება). ასევე ზოგიერთი გრაფიკული ნიმუშის ნახვაა შესალებელი example(persp) ბრძანებითაც.
